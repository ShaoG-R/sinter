# Sinter

**下一代高性能 Web 内容编译器**

> 🚧 **项目目前处于 Alpha 开发阶段**。架构与 API 可能会在 v1.0 之前发生重大变更。

Sinter 是一个基于 **Rust** 和 **WebAssembly** 构建的全栈式静态站点生成系统。不同于传统的 SSG (如 Hexo, Hugo) 生成静态 HTML 集合，Sinter 编译生成的是一个极致轻量、瞬间响应的 **单页应用 (SPA)**。

它旨在解决传统静态站点在文章数量增长后的构建速度瓶颈，以及 Web 页面在交互体验上的局限性。

## 🌟 核心特性

-   **自研 Sinter UI 引擎**:  
    基于 **No VDOM** 和 **细粒度响应式 (Fine-Grained Reactivity)** 设计。直接操作 DOM，消除了 Diff 算法的运行时开销。完整运行时（含 UI 引擎、路由、主题逻辑）仅约 **337KB** (未压缩)。

-   **智能分片加载 (Sharded Loading)**:  
    采用 App Shell 架构。首屏仅加载极轻量的元数据，文章详情与列表数据分离并**按需异步加载**。无论站点包含 10 篇还是 10,000 篇文章，首屏加载时间 (FCP) 始终保持在毫秒级。

-   **原生级交互体验**:  
    内置基于 History API 的**原生路由系统**，页面切换无刷新、无白屏。支持丰富的动态交互，体验接近原生应用。

-   **并行编译架构**:  
    编译器核心利用 Rayon 实现极致的数据并行处理，构建速度随 CPU 核心数线性增长。

## ⚡️ 生态位对比

| 特性 | Sinter | Hugo / Hexo |
| :--- | :--- | :--- |
| **运行时** | **Rust + WASM** | Go / Node.js |
| **产物形态** | **SPA (单页应用)** | 静态 HTML 集合 |
| **体验** | **无刷新瞬开**，原生级交互 | 页面跳转需重载 |
| **部署成本** | **O(1)** (体积不随文章数膨胀) | O(N) (随文章数线性增长) |
